ADR 0008: Использование API-шлюза с публикацией OpenAPI

Дата: 2024-07-10

## Контекст

Для улучшения взаимодействия между микросервисами и обеспечения единого входа в систему необходим API-шлюз. 
Также необходимо публиковать спецификации OpenAPI для упрощения интеграции смежных сервисов.

## Решение

Внедрить API-шлюз, который будет выполнять маршрутизацию запросов, 
обеспечивать безопасность и публиковать спецификации OpenAPI для всех микросервисов.

### Причины

1. **Единая точка входа**: Упрощение управления запросами через единый API-шлюз.
2. **Безопасность**: Улучшение безопасности за счет единой точки контроля доступа.
3. **Документация**: Публикация спецификаций OpenAPI для упрощения интеграции и взаимодействия между сервисами.
4. **Мониторинг и логирование**: Централизованное управление логированием и мониторингом запросов.

## Статус

Принято

## Последствия

Упрощение взаимодействия между микросервисами, 
улучшение безопасности и предоставление актуальной документации для разработчиков смежных сервисов.
